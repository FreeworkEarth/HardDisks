## 2026-05-23: Visual Defaults and Cochlea Analogy

Current SDL2 defaults:

- MB speed colouring is on by default.
- Global x-density curve overlay is on by default.
- `MB scale SET` still manually captures the current speed range; otherwise the colour scale uses a fixed thermal reference.

Useful physical analogy:

The live wall/spring system is not literally an ear, but it can be treated like a minimal coupled-resonator / transducer chain.

- gas particles carry pressure and density waves through the medium
- internal walls respond to local pressure impulses
- wall mass and spring stiffness set response times and resonance
- spring/load energy is a proxy for transmitted mechanical signal
- a future bank of walls with different masses/springs could act like a crude frequency-selective sensor array

Ear analogy, phrased carefully:

- ossicles transmit mechanical vibration into the cochlea
- the cochlea separates frequencies through position-dependent mechanical response
- hair cells transduce local motion into neural signals
- our walls/springs can be viewed as artificial mechanical sensors for pressure-wave structure, but not as a biological cochlea model yet

This makes the instrument idea coherent: use physical observables as modulation sources.

- pressure / collision rate -> filter cutoff or grain density
- spring energy -> feedback, resonance, reverb, distortion
- wall displacement -> pitch, wavetable position, pan
- density contrast -> texture brightness or stereo width
- MB temperature/speed spread -> noise colour or saturation

## 2026-05-23: Scene Reset Geometry Fix

The F3 orange scene-reset panel now treats the energy-transfer preset like the production runs:

- `spring L0` is an applied/rebuild parameter.
- For energy transfer, wall 0 is placed at `spring L0`.
- With 2 walls and `spring L0=20`, the walls are `20σ, 30σ`.
- The spring equilibrium is set to the same `spring L0`, so the spring starts unloaded.
- Eta is computed from the actually occupied compartments, not from the empty left compartment.
- The orange panel can switch the rebuild engine between `EDMD`, `HYBRID PP-EDMD`, and `TIME+TOI`; Szilard remains locked to EDMD because the semipermeable gates live there.

For energy transfer with the left compartment empty:

`eta_s = N_s*pi*r^2 / (W_s*H)`

where `N_s` is the particle count in active segment `s`, `r` is the physical collision radius, `W_s` is that segment width in sigma units, and `H` is the box height in sigma units.

Important visual caveat:

If eta is fixed and particle number increases, the physical radius decreases like `1/sqrt(N)`. A 4000-particle run at the same eta can still look much denser than a 200-particle run because there are many more dots and because the SDL draw-min-radius slider can make tiny particles visible larger than their physical radius.

## 2026-05-23: TIME+TOI Realtime Backend

The F3 engine switch now labels the time-integrated backend as `TIME+TOI`.

Why:

- Plain TIME with `sub=1` missed too many collisions after the first pressure wave.
- Missed collisions created deep overlaps.
- The old overlap repair then pushed particles into dense sheets, which looked like artificial clamping.
- EDMD did not show this because EDMD resolves hard-disk events exactly.

What changed:

- Applying `TIME+TOI` enables particle-particle time-of-impact correction.
- Applying `TIME+TOI` enables adaptive substeps with a minimum of 4.
- The spring kick is integrated inside the same substep loop as wall collisions.
- `update_wall()` no longer moves the spring wall a second time after the particle/wall substep solver already moved it.

This is still an approximate realtime backend. EDMD remains the ground truth for scientific plots. `TIME+TOI` is the practical high-N / SDL / instrument backend.

## 2026-05-23: Hybrid PP-EDMD Live Engine

The F3 orange panel now has three engine choices:

- `EDMD`: full event-driven hard-disk solver, most trustworthy, slowest at high particle number.
- `HYBRID PP-EDMD`: particle-particle collisions use EDMD, while walls, pistons, and spring remain live time-controlled.
- `TIME+TOI`: fastest realtime backend, approximate, useful for dense visual/instrument exploration.

This hybrid mode is the first practical mixed simulation between time integration and EDMD. It should reduce particle-particle overlap clamping compared with `TIME+TOI`, while still keeping live spring/piston controls responsive.

The CLI aliases are now:

```bash
--mode=edmd
--mode=hybrid
--mode=edmd-hybrid
--mode=event-split
--mode=time
```

Important naming caveat:

`HYBRID PP-EDMD` is not yet the final fully symmetric event-split integrator from the research note. It is the already implemented mixed backend: exact EDMD for particle-particle events, time/CCD for moving boundaries.

## 2026-05-23: High-N Eta Visual Fix

The default render-only minimum particle radius is now `1.0px` instead of `1.5px`.

When applying a new F3 scene, the draw-min radius is also clamped down to the physical collision radius if the old visual minimum would inflate the displayed particle size. This matters for high particle counts:

- eta is physical packing fraction
- the collision radius decreases like `1/sqrt(N)`
- a visual minimum radius can make `eta=0.2` look like a much higher eta

The HUD now prints both:

- physical radius in sigma and pixels
- draw radius in pixels

So if a high-N scene still looks too dense, first check whether `draw` is larger than the physical pixel radius.
